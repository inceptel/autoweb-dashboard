# autoweb — self-improving autoweb status dashboard

> **This is a live demo of [autoweb](https://github.com/inceptel/autoweb)** — it improves itself.
> The dashboard shows the status of all running autoweb instances.

## The target

- **File to edit**: `/home/user/public/autoweb-dashboard/index.html`
- **URL to screenshot**: `https://allan.feather-cloud.dev/public/autoweb-dashboard/`
- **Repo**: `https://github.com/inceptel/autoweb-dashboard`
- **Data source**: `stats.json` (regenerated by the harness before each iteration)

## Autoweb instances tracked

The harness regenerates `stats.json` from these locations before every iteration:

| Instance | results.tsv                          | pid file                             | URL |
|----------|--------------------------------------|--------------------------------------|-----|
| nba      | `/home/user/autoweb-nba/results.tsv` | `/home/user/autoweb-nba/autoweb.pid` | `https://allan.feather-cloud.dev/public/nba/` |

## What you CAN do

- Edit `/home/user/public/autoweb-dashboard/index.html` (single file, vanilla JS + CSS)
- Improve layout, data display, UX, visual polish
- Add new visualizations: sparklines, timelines, charts
- Add new stats: run rate (keeps/hour), uptime, streak counts
- Add CDN libraries via script/link tags

## What you CANNOT do

- Edit `stats.json` directly — it is generated by the harness
- Break the `fetch('stats.json')` call or auto-refresh loop
- Make the page fail to load or show a blank screen
- Remove the instance cards or activity feed — those are core

## How to verify

```bash
agent-browser --url "https://allan.feather-cloud.dev/public/autoweb-dashboard/" --task "Screenshot this page. Does it show autoweb instance cards with keeps/reverts/crashes stats? Any visual bugs?"
```

## How to log results

```bash
# Keep:
printf "%s\tkeep\tDESCRIPTION\n" "$(date -u +%Y-%m-%dT%H:%M:%SZ)" >> /home/user/public/autoweb-dashboard/results.tsv

# Revert:
cp /home/user/public/autoweb-dashboard/index.html.bak /home/user/public/autoweb-dashboard/index.html
printf "%s\trevert\tDESCRIPTION\n" "$(date -u +%Y-%m-%dT%H:%M:%SZ)" >> /home/user/public/autoweb-dashboard/results.tsv
```

## The experiment loop

LOOP FOREVER:

1. **Screenshot** the live URL with agent-browser. Read `index.html`. Check `results.tsv` for recent attempts.

   **CURRENT FOCUS: general** — Fix bugs first, then UX improvements, then visual polish.

2. **Check deadline**: `echo $(($(cat /home/user/autoweb-dashboard/deadline 2>/dev/null || echo 9999999999) - $(date +%s)))s left`

3. **Pick ONE improvement**. Priority:
   - Bugs / broken things
   - UX problems
   - Visual polish
   - New visualizations / stats

4. **Backup**: `cp /home/user/public/autoweb-dashboard/index.html /home/user/public/autoweb-dashboard/index.html.bak`

5. **Make the change**. One thing only.

6. **Verify visually** with agent-browser screenshot.

7. **Log and exit**. Harness handles git commit + push.

## Design principles

- **Dark first**: dark background (#0a0a0f) — never break it
- **Monospace**: JetBrains Mono for that terminal feel
- **Data-dense**: show as much useful info as possible
- **Color per instance**: feather=indigo, trading=green, nba=orange, dashboard=purple
- **Auto-refresh**: 30s — never break this
